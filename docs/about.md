# About agent-face

agent-face は、AI コーディングエージェントの状態を ASCII アニメーションで可視化する **CLI アプリケーション**である。別ターミナルで起動し、エージェントの動作に連動して表情がリアルタイムに変化する。

---

## なぜこのツールを作るのか

AI コーディングエージェント（Claude Code、Codex 等）は強力だが、作業中の体験は退屈で無機質だ。

- エージェントが思考している間、人間はただ待つしかない
- ターミナルに流れるテキストには「感情」がない
- 今エージェントが何をしているのか、直感的に分からない

人間はビジュアルを持った存在に好感を持つ。ノベルゲームやギャルゲーのキャラクターが場面に応じて表情を変えるように、AI エージェントにも「顔」があれば、開発体験はもっと楽しくなる。

**agent-face は、AI エージェントに顔を与えるツールである。**

---

## 体験のビジョン

### エージェントが「いる」と感じられる開発環境

```
┌─ Terminal 1: Claude Code ────────┐  ┌─ Terminal 2: agent-face ──┐
│                                  │  │                           │
│  > Fix the login bug             │  │       /\_/\               │
│                                  │  │      ( ●  ● )    thinking │
│  I'll look at the auth module... │  │       ( ω  )  ·           │
│                                  │  │         ~~~   · ·         │
│  Reading src/auth.rs...          │  │                           │
│                                  │  │                           │
└──────────────────────────────────┘  └───────────────────────────┘
```

エージェントがコードを読んでいるとき、考え込んでいるとき、書き出したとき、エラーに遭遇したとき — 顔が変わる。テキストログを目で追わなくても、エージェントの状態が一目で分かる。

### ノベルゲームの立ち絵のように

ノベルゲームでは、キャラクターの表情がシーンの感情を伝える。プレイヤーはテキストを読む前に、表情だけで状況を察する。

agent-face はこれと同じ原理を開発環境に持ち込む:

| エージェントの状態 | 顔の表現 | 人間が受け取る印象 |
|---|---|---|
| 待機中 | 穏やかな表情、まばたき | 「準備できてるよ」 |
| 思考中 | 目が動く、泡パーティクル | 「考えてるな」 |
| 書き込み中 | 集中した目、プログレスバー | 「書いてる、順調だ」 |
| エラー | 困った顔 | 「何か問題が起きた」 |
| 成功 | 嬉しそうな顔、星パーティクル | 「できた！」 |
| 入力待ち | 耳を傾ける表情 | 「何か聞きたいみたい」 |

### 自分だけの「相棒」を選べる

キャラクターは外部ファイルで定義される。猫、ロボット、ドット絵 — 好きなキャラクターファイルを作れば、自分だけの相棒と開発できる。コミュニティがキャラクターを共有すれば、選択肢はさらに広がる。

## 導入体験

### 気軽さが最優先

agent-face は「試してみよう」と思った人が数分で動かせるべきである。

```bash
# インストール
cargo install agent-face

# 即座に動く — 設定ファイルもキャラクターファイルも不要
agent-face
```

初回起動でデフォルトキャラクター（cat）が MANUAL モードで動く。キーボードで表情を切り替えて遊べる。これだけで「おっ、かわいい」という第一印象を作る。

### エージェントとの連携: 一度セットアップすれば全プロジェクトで動く

エージェントとの連携はグローバル設定で行う。プロジェクトごとのセットアップは不要。

```bash
# セットアップ（PC につき一度だけ）
agent-face setup claude-code
```

このコマンドが行うこと:
- `~/.config/agent-face/adapters/claude-code/face-state.sh` を配置
- `~/.claude/settings.json` にフック設定を追記

以降、どのプロジェクトでも `claude` コマンドを実行すれば agent-face が連動する。

```bash
cd ~/project-a && claude   # agent-face が連動する
cd ~/project-b && claude   # ここでも連動する
```

### 起動方法: 手動と自動

**手動起動（デフォルト）**

ユーザーが別ターミナルで `agent-face` コマンドを実行する。

```bash
# ターミナル 1
agent-face

# ターミナル 2
cd ~/myproject && claude
```

**自動起動**

エージェント起動時に agent-face が新規ターミナルウィンドウで自動的に立ち上がる。アダプターの hook スクリプトが起動を担当する（agent-face 本体はこのロジックを持たない）。

`~/.config/agent-face/config.toml` で切り替える:

```toml
auto_launch = true
```

設定ファイルが存在しなければすべてデフォルト値で動作する。`~/.config/agent-face/` ディレクトリはインストール時には作成されず、ユーザーが `agent-face setup` を実行するか、手動で設定をカスタマイズするときに初めて作成される。

### 段階的なカスタマイズ

ユーザーは自分のペースで深く使っていける:

1. **インストール直後** — デフォルトキャラクターで動く。設定不要
2. **エージェントと連携** — アダプターをプロジェクトにコピー
3. **自動起動を有効化** — `~/.config/agent-face/config.toml` を作成
4. **キャラクターを選ぶ** — コミュニティが作ったキャラクターをダウンロードして配置
5. **自分で作る** — キャラクター定義ファイルを書いて自分だけの相棒を作る
6. **共有する** — 作ったキャラクターを公開して他の人にも使ってもらう

## キャラクターシステムのビジョン

### Phase 1: ASCII アート（初期リリース）

ターミナル上の ASCII アートで表情を表現する。テキストベースなので軽量で、どの環境でも動く。

### Phase 2: 画像・GIF 対応（将来）

ASCII アートに留まらず、任意の画像や GIF アニメーションをキャラクターとして使えるようにする。ノベルゲームの立ち絵そのものを再現できる世界。

```
キャラクターパック例:
  my-character/
    idle.gif
    thinking.gif
    writing.gif
    error.gif
    success.gif
    listening.gif
    character.toml   ← メタデータ（名前、速度、表示設定）
```

### 設計原則: バイナリに閉じ込めない

キャラクターデータはバイナリに埋め込まない。理由:

- **拡張性**: ASCII アート → 画像 → GIF → 将来の未知のフォーマット。バイナリに埋め込むと、フォーマット追加のたびに再コンパイルが必要になる
- **コミュニティ**: ユーザーがキャラクターを自由に作成・共有できる文化を作りたい。バイナリの外にあることが前提
- **サイズ**: 画像/GIF を含むキャラクターパックはバイナリに埋め込むには大きすぎる

ただし「インストール直後に動く」体験は必須なので、デフォルトキャラクターの扱いは工夫が要る:

| 方式 | 即座に動く | バイナリ非依存 | 備考 |
|---|---|---|---|
| バイナリ埋め込み | Yes | No | 将来の画像対応と矛盾 |
| インストール時にファイルを配置 | Yes | Yes | `cargo install` では制御しにくい |
| 初回起動時に自動生成 | Yes | Yes | バイナリにテンプレートは持つが、すぐ外部化される |
| バイナリ埋め込み + 外部上書き | Yes | 部分的 | 外部ファイルがあればそちらを優先。なければ組み込みにフォールバック |

## 誰のためのツールか

- AI コーディングエージェントを日常的に使う開発者
- 作業中の待ち時間をもっと楽しくしたい人
- 開発環境に個性や遊び心を持ち込みたい人

## 何を目指さないか

- 生産性ツール — agent-face はエージェントの性能を上げるものではない
- モニタリングツール — ログや統計を表示するものではない
- GUI アプリケーション — ターミナルの中で完結する（Phase 1）

agent-face は「体験」のためのツールである。
