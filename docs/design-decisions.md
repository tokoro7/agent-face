# Design Decisions

これまでの議論で決定・確認した設計上の判断を記録する。

| # | 問題 | 決定 |
|---|------|------|
| 01 | Cargo.lock を gitignore すべきか | コミットする。バイナリアプリケーションの慣習に従う |
| 02 | state file の書き込み方式 | アトミック書き込み（temp file + rename） |
| 03 | シグナルハンドリング・ターミナル復元 | 実装する。正常終了・SIGINT・SIGTERM・パニックすべてで復元 |
| 04 | agent-face の形態 | フォアグラウンドで動く CLI アプリケーション |
| 05 | フック設定のスコープ | グローバル（`~/.claude/settings.json`）。プロジェクトごとのセットアップ不要 |
| 06 | エージェント連携のセットアップ | `agent-face setup claude-code` で一度だけ行う |
| 07 | 起動方法 | 手動（デフォルト）と自動の2モード。config で切り替え |
| 08 | 自動起動の実装場所 | アダプターの hook スクリプトが担当。agent-face 本体は関知しない |
| 09 | config ディレクトリの作成タイミング | インストール時には作らない。setup コマンドまたは手動カスタマイズ時に初めて作成 |
| 10 | キャラクターデータの埋め込み | バイナリに埋め込まない方針（将来の画像/GIF対応を見据えて）。配布方法は未決定 |
| 11 | 非同期ランタイム | 使わない。スレッドベースの並行処理 |
| 12 | セットアップ済み判定 | ファイル存在ではなく config.toml のフラグ (`claude_code_setup`) で管理 |
